Исходный код перемножения матриц из Лабораторной работы №1 был модифицировандля работы с CUDA C++. Были проведены тесты для разных конфигураций сетки блоков и размеров матриц. Для автоматизации проведения тестов используется скрипт `run_lab.py`, который запускает скомпилированный `Lab4.exe` с разными параметрами.

Матрицы A, B, C размером N×N выделяются на GPU через функцию `cudaMalloc`. Заполнение исходных матриц происходит на Host (процессоре) чтением из текстового файла, после чего копируется в видеопамять (Device) через `cudaMemcpyHostToDevice`. Тестировались 6 конфигураций блоков. Сетка (Grid) высчитывается динамически: `grid = (N+dim_x-1)/dim_x`. `matrixMulKernel<<<numBlocks, threadsPerBlock>>>(d_A, d_B, d_C, N)` — каждый поток вычисляет отдельную ячейку результирующей матрицы. Сама математика внутри ядра реализована стандартным циклом `for`. Для максимально точного тайминга использовались `cudaEvent_t`.

## Выводы
Из результатов видно, что GPU превосходит CPU в матричных задачах на порядки благодаря тысячам CUDA-ядер (для сравнения лучший результат на CPU на 8 потоках матрицы 1600ч1600 - 995 мс, а на GPU - 12 мс). Блок 8x8 (64 потока) стабильно показывает худшее время на больших матрицах (15.2647 мс на 1600x1600). Такого количества потоков недостаточно, чтобы планировщик мог эффективно скрывать задержки при обращении к глобальной памяти.
